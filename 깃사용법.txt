1. 작업 전 할 일
git checkout model                    # model 브랜치로 이동 (각 작업할 브랜치로 이동하는거임) 만약 web브런치로 이동하고 싶으면 model대신 web치기
git pull origin model                 # 원격 최신 코드 가져오기(팀원들이(브랜치 내용만) 한 코드 가져오기 이것도 web으로 이동하고 싶다면 web으로 변경)
------------------------------------------------------------------------------------------------------------------------------------------------------------------------
2. 내가 작업한거 올리기
git status                            # 변경된 파일 확인(내 작업 상태 확인)
git add .                             # 전체 변경사항 추가 (또는 git add 파일명)
------------------------------------------------------------------------------------------------------------------------------------------------------------------------
5. 작업 내용 커밋
git commit -m "🔥 모델 학습 기능 구현 완료"  # 명확한 커밋 메시지 작성(" "<-여기안에 제목을 정하기)
------------------------------------------------------------------------------------------------------------------------------------------------------------------------
6. 내 작업 결과 원격 GitHub에 푸시
git push origin model    <-이거까지하면 내가 한거 model브랜치에 반영됨 #만약 web에서 했다면 model대신 web 
# -----------------------------------------------

# 7. 작업 끝나고 main에 반영 (테스트용 브랜치로 병합)
# 이 작업은 팀원과 상의 후 진행해야 안전!
# -----------------------------------------------
git checkout main                      # main 브랜치로 이동

git pull origin main                    # main 최신화
git merge model                       # model 병합 (만약 web을 하고 싶다면 web으로)
git push origin main                  # 깃허브에 병합된 main 푸시
이거는 우리가 각 브랜치에 작업한거 main브랜치에 하나로 합쳐 제대로 작동하는지 확인가능


# -----------------------------------------------
# 8. 통합 테스트 성공 시, 배포용 master 브랜치에 병합
# -----------------------------------------------
git checkout master                   # 배포 브랜치 이동
git pull origin master                # master 최신화
git merge main                        # main 병합
git push origin master                # 깃허브에 푸시
이거는 실제 모든 프로젝트 과정이 끝나고 제출 전 마지막 코드를 올리는거(완성본)
